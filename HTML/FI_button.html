Element Button

Digunakan untuk membuat tombol yang bisa diklik pada halaman web. Ini memungkinkan pengguna untuk melakukan berbagai tindakan saat tombol diklik. Berikut contoh penggunaannya:



Attribut button

  • < button autofocus >Klik Saya< /button>

Digunakan untuk menentukan apakah tombol tersebut harus mendapatkan fokus secara otomatis ketika halaman dimuat.

  • < button disabled >Klik Saya< /button>

Digunakan untuk menonaktifkan atau membuat tombol tidak dapat diakses atau di klik oleh pengguna. Ketika atribut ini ditambahkan, tombol akan tetap ada di halaman, tetapi pengguna tidak akan bisa melakukan interaksi dengannya.

Disabled juga bisa diatur melalui JavaScript.

  • < form id="myForm" action="/submit" method="post">
    < input type="text" name="username">
    < button form="myForm">Kirim
    < /form>

Digunakan untuk menentukan formulir mana yang akan di-submit ketika tombol tersebut ditekan. Ini berguna ketika tombol tersebut berada di luar formulir yang ingin Anda kirimkan.

form juga dapat diatur secara dinamis melalui JavaScript. Misalnya:

  • < form id="myForm" action="/default_action" method="post">
    < input type="text" name="username">
    < button form="myForm" formaction="/custom_action">Kirim< /button> < /form>

Digunakan untuk menentukan URL atau lokasi yang akan dituju ketika tombol tersebut ditekan, menggantikan aksi formulir yang telah ditentukan.







  • < button form="myForm" formenctype="multipart/form-data">Submit< /button>

Digunakan untuk menentukan jenis enkripsi yang akan digunakan saat data formulir dikirimkan ke server melalui tombol tersebut.

application/x-www-form-urlencoded: Ini adalah jenis default dari enkripsi formulir. Data formulir dikirimkan sebagai string URL-encoded, di mana karakter non-alphanumeric dikodekan sebagai persentase dari ASCII. Ini cocok untuk mengirimkan data formulir yang sederhana.

multipart/form-data: Ini digunakan ketika formulir mengandung file yang akan diunggah. Data formulir dikirimkan dalam beberapa bagian terpisah, termasuk bagian untuk file-file yang diunggah. Ini cocok untuk pengiriman formulir yang mengandung data biner, seperti gambar, file audio, atau file video.

text/plain: Ini adalah jenis enkripsi yang paling sederhana. Data formulir dikirimkan sebagai teks biasa, tanpa ada encoding tambahan. Ini cocok untuk situasi di mana Anda ingin data formulir mudah dibaca oleh manusia, seperti ketika melakukan debug atau pengujian.





  • < form id="myForm" action="/submit" method="post">
    < input type="text" name="username">
    < button form="myForm" formmethod="get">Submit (GET)< /button>
    < button form="myForm" formmethod="post">Submit (POST)< /button>
    < /form>

Menentukan metode HTTP yang akan digunakan saat formulir yang terkait dengan tombol tersebut dikirimkan. Ini memungkinkan Anda untuk menentukan metode pengiriman formulir yang berbeda tergantung pada tombol mana yang ditekan pengguna.

GET:
- Data dikirimkan sebagai bagian dari URL.
- Data terlihat dalam URL, karena disertakan di dalam string query setelah tanda tanya (?).
- Jumlah data yang dapat dikirimkan terbatas karena keterbatasan panjang URL.
- Cocok untuk permintaan yang tidak mengubah data di server (misalnya, pencarian atau permintaan halaman).
- Data dapat disimpan dalam cache browser atau ditandai sebagai bookmark.

POST:
- Data dikirimkan dalam tubuh permintaan HTTP.
- Data tidak terlihat di URL.
- Tidak ada batasan panjang data yang dapat dikirimkan.
- Cocok untuk permintaan yang mengubah atau menambahkan data di server (misalnya, pengiriman formulir atau pengiriman data sensitif).
- Data tidak disimpan dalam cache browser dan tidak bisa ditandai sebagai bookmark.







  • < button form="myForm" aria-busy="formnovalidate ">Submit< /button>

Digunakan untuk menonaktifkan validasi formulir Saat tombol ditekan, formulir akan dikirimkan ke server tanpa melalui validasi yang biasanya dilakukan oleh browser.




Digunakan untuk menentukan di mana respons dari pengiriman formulir akan ditampilkan setelah formulir dikirimkan. Ini memungkinkan Anda untuk menentukan apakah respon akan ditampilkan di jendela baru, jendela yang sama, atau di suatu iframe.

1 _self: Respon akan ditampilkan di jendela atau tab yang sama di mana formulir itu sendiri berada (default)
2 _blank: Respon akan ditampilkan di jendela atau tab baru.
3 _parent: Respon akan ditampilkan di frame parent dari frame tempat formulir itu berada (jika ada).
4 _top: Respon akan ditampilkan di jendela yang sama seperti _blank, tetapi jika formulir ada di dalam frame, itu akan membuat respon ditampilkan di jendela paling atas.
5 iframe_name: Respon akan ditampilkan di dalam iframe dengan nama yang ditentukan.







  • < button type="submit" name="submit_button" value="cancel">Cancel< /button>

Digunakan untuk memberikan nama kepada tombol ketika data formulir dikirimkan ke server

  • < button type="submit">Submit< /button>

Digunakan untuk menentukan tipe dari sebuah tombol. Nilai yang umum untuk atribut ini adalah "submit", "reset", dan "button".

  • < button type="submit" value="save">Save< /button>

Digunakan untuk menentukan teks atau nilai yang akan dikirimkan ke server saat tombol diklik dalam sebuah formulir. Nilai ini sering kali digunakan bersama dengan atribut type="submit" untuk menentukan teks yang akan muncul pada tombol submit.
