Element form

Tag digunakan untukmembuat formulir HTML sebagai input pengguna,

Contoh:
Formulir html dengan dua bidang input dan satu tombol kirim







Klik tombol "Kirim" dan data formulir akan dikirim ke halaman di Server disebut "action_page.php".

Attribut form

  • < form action="/submit-form" method="post" accept-charset="UTF-8"> < /form>

Digunakan untuk menentukan karakter encoding yang akan digunakan saat data formulir dikirimkan ke server. Karakter encoding adalah cara untuk mewakili karakter teks dalam bentuk biner yang dapat dipahami oleh komputer.

  • < form action="/submit-form.php" method="post">< /form>

Digunakan untuk menentukan URL atau alamat ke mana data formulir akan dikirimkan setelah formulir tersebut disubmit.

  • < form action="/submit-form" method="post" autocomplete="off" / "on"> < /form>

Digunakan untuk mengontrol apakah browser harus menyediakan fitur pengisian otomatis (autofill) untuk formulir. Fitur autofill ini memungkinkan browser untuk mengingat nilai-nilai yang dimasukkan pengguna pada formulir sebelumnya dan menawarkan opsi untuk mengisi formulir dengan nilai-nilai tersebut secara otomatis saat pengguna mengunjungi kembali halaman yang sama atau serupa.






  • < form action="/submit-form" method="post" enctype="multipart/form-data"> < /form>

Digunakan untuk menentukan cara encoding (pengkodean) dari data yang akan dikirimkan ketika formulir disubmit. Ini terutama relevan ketika metode POST digunakan untuk mengirimkan data formulir yang mungkin mengandung file.






1. application/x-www-form-urlencoded: Ini adalah nilai default. Data formulir dikodekan dalam format URL-encoded sebelum dikirim. Ini berarti spasi dikodekan sebagai +, dan karakter khusus lainnya dikodekan sebagai %xx, di mana xx adalah representasi heksadesimal dari kode karakter ASCII.
2. multipart/form-data: Biasanya digunakan ketika formulir mengandung elemen input type="file". Data formulir dan file yang dipilih akan dikirimkan sebagai bagian-bagian terpisah. Ini memungkinkan file-file untuk dikirimkan melalui HTTP POST.
3. text/plain: Data formulir dikirimkan dalam format teks mentah tanpa pengkodean tambahan. Ini lebih jarang digunakan dan tidak sering disarankan, karena pengkodean data dalam bentuk URL-encoded atau multipart/form-data biasanya lebih berguna.

  • < form action="/submit-form" method="post" / "get"> < /form>

Digunakan untuk menentukan metode HTTP yang akan digunakan saat mengirimkan data formulir ke server. Metode yang paling umum digunakan adalah GET dan POST.






1. GET: Metode ini mengirimkan data formulir sebagai bagian dari URL. Data dikirim sebagai query string yang dapat dilihat oleh pengguna dan disimpan dalam riwayat peramban. Metode GET biasanya digunakan untuk permintaan yang sederhana dan ketika tidak sensitif terhadap keamanan atau privasi, seperti pencarian.
2. POST: Metode ini mengirimkan data formulir sebagai bagian dari tubuh permintaan HTTP. Data tidak ditampilkan di URL dan tidak tersimpan di riwayat peramban. Metode POST sering digunakan ketika pengguna mengirimkan data sensitif atau data yang lebih besar.

  • < form action="/submit-form" method="post" name="registrationForm">
    < input type="text" name="username">
    < input type="password" name="password"> < /form>

Digunakan untuk memberi nama pada formulir. Nama ini digunakan untuk mengidentifikasi formulir saat data dikirimkan ke server. Setiap elemen formulir (seperti input, select, atau textarea) juga memiliki atribut name yang digunakan untuk memberi nama pada elemen tersebut. Ketika formulir disubmit, data dari setiap elemen formulir akan dikirimkan ke server dengan menggunakan nama elemen tersebut.

Dalam contoh di atas, elemen form diberi nama "registrationForm", dan dua elemen input diberi nama "username" dan "password" masing-masing. Ketika formulir ini disubmit, data dari input "username" dan "password" akan dikirimkan ke server dengan menggunakan nama tersebut.






  • < form action="/submit-form" method="post" novalidate > < /form>

adalah atribut boolean yang digunakan untuk menonaktifkan validasi bawaan browser pada formulir sebelum pengirimannya ke server.

Secara default, browser akan melakukan validasi terhadap input pada formulir sebelum mengizinkan pengirimannya ke server. Validasi ini mencakup pemeriksaan apakah input yang diberikan oleh pengguna sesuai dengan aturan yang ditentukan, seperti validasi keharusan, tipe data, atau pola tertentu.

Namun, dengan menambahkan atribut novalidate pada elemen < form>, Anda dapat mematikan validasi bawaan browser ini. Ini berguna jika Anda ingin melakukan validasi secara khusus di sisi server atau menggunakan mekanisme validasi yang disediakan oleh JavaScript.




  • < form action="/submit-form" method="post" target="_blank" / "_self" / "_parent" / "_top"> < /form>

digunakan untuk menentukan di mana respon dari pengiriman formulir akan ditampilkan setelah formulir tersebut disubmit

1. _blank: Respon dari pengiriman formulir akan ditampilkan di jendela atau tab baru.
2. _self: Respon dari pengiriman formulir akan ditampilkan di jendela atau browsing context yang sama. Ini adalah perilaku default jika target tidak ditentukan.
3. _parent: Respon dari pengiriman formulir akan ditampilkan di browsing context yang mengandung elemen iframe, atau di jendela induk jika formulir tersebut berada dalam iframe.
4. _top: Respon dari pengiriman formulir akan ditampilkan di jendela paling atas (top-level browsing context).
