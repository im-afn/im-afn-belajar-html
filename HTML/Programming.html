Element Script

Tag script> dalam HTML digunakan untuk menyisipkan atau merujuk pada kode eksekusi, biasanya JavaScript, di dalam dokumen HTML.

<< script src="path/to/your/script.js"> < /script>

<< script> console.log('Skrip inline dijalankan'); < /script>

Letakkan tag script> di akhir tag body> jika tidak menggunakan async atau defer untuk meningkatkan kinerja halaman.

Gunakan skrip eksternal untuk memudahkan pemeliharaan dan memanfaatkan caching browser

Hindari penggunaan skrip inline jika memungkinkan untuk menjaga struktur HTML tetap bersih dan menjaga pemisahan antara konten dan logika.

Attribut scirpt

  • << script src="demo_async.js" async> < /script>

Atribut async pada tag script> dalam HTML digunakan untuk menginstruksikan browser agar mengunduh skrip secara asinkron dengan proses pemuatan halaman. Ini berarti bahwa skrip akan diunduh dan dieksekusi tanpa memblokir penguraian halaman HTML, memungkinkan konten halaman untuk dimuat lebih cepat.

async: Ketika atribut async digunakan, skrip akan diunduh bersamaan dengan pemuatan halaman dan dieksekusi segera setelah unduhannya selesai, tanpa menunggu pemuatan seluruh halaman selesai. Hal ini cocok untuk skrip yang tidak bergantung pada DOM atau skrip lain.

Paragraf ini akan ditampilkan sebelum skrip selesai diunduh dan dieksekusi.

  • << script src="https://example.com/script.js" crossorigin="anonymous"> < script />

Atribut crossorigin pada tag script> dalam HTML digunakan untuk mengontrol bagaimana permintaan skrip eksternal menangani lintas asal (cross-origin). Ini penting untuk situasi di mana skrip diambil dari domain berbeda dan diperlukan kontrol yang lebih besar terkait dengan keamanan dan kebijakan sumber daya bersama (CORS).

anonymous: Meminta agar permintaan dilakukan tanpa kredensial (seperti cookie, sertifikat HTTP, atau header otorisasi). Ini adalah pengaturan default yang memungkinkan browser untuk mendapatkan skrip dari domain lain, tetapi tanpa mengirimkan informasi pengenal pengguna.

use-credentials: Meminta agar permintaan dilakukan dengan mengirimkan kredensial. Ini berarti cookie dan header otorisasi akan dikirimkan dengan permintaan. Server harus mengatur header Access-Control-Allow-Credentials menjadi true untuk mengizinkan hal ini.

Menggunakan crossorigin="anonymous": Saat Anda memuat skrip dari domain lain dan tidak ingin atau tidak perlu mengirimkan kredensial pengguna. Contohnya, memuat pustaka JavaScript seperti jQuery dari CDN.

Menggunakan crossorigin="use-credentials": Saat skrip membutuhkan akses ke sumber daya yang memerlukan kredensial, misalnya, jika skrip memerlukan otorisasi pengguna.

Dengan menggunakan atribut crossorigin secara tepat, Anda dapat mengelola keamanan dan kinerja aplikasi web Anda saat memuat skrip dari domain eksternal.

  • << script src="demo_defer.js" defer < /script>

Atribut defer pada tag script> dalam HTML digunakan untuk menunda eksekusi skrip sampai dokumen HTML sepenuhnya diuraikan (di-parsing). Skrip yang menggunakan defer akan tetap diunduh bersamaan dengan pemuatan halaman, tetapi akan dieksekusi hanya setelah seluruh konten halaman selesai diuraikan. Ini memungkinkan HTML untuk dimuat lebih cepat dan memastikan bahwa skrip tidak mengganggu proses penguraian halaman.

defer: Ketika atribut defer digunakan, browser akan menunda eksekusi skrip sampai dokumen HTML sepenuhnya diuraikan. Atribut ini hanya berlaku untuk skrip eksternal dan diabaikan pada skrip inline.

  • << script src="https://code.jquery.com/jquery-3.3.1.slim.min.js" integrity="sha384-q8i/X+965DzO0rT7abK41JStQIAqVgRVzpbzo5smXKp4YfRvH+8abtTE1Pi6jizo" crossorigin="anonymous"> < /script>

Atribut integrity pada tag script> dalam HTML digunakan untuk memastikan bahwa konten skrip yang diambil dari server tidak dimodifikasi dan sesuai dengan versi yang diharapkan. Ini dilakukan dengan menggunakan nilai hash kriptografis yang dikodekan dalam base64. Jika skrip yang diunduh tidak cocok dengan hash yang diberikan, maka browser akan menolak untuk menjalankannya. Ini membantu melindungi terhadap serangan penyisipan konten berbahaya (content injection attacks).

Skrip dari CDN: Sangat disarankan menggunakan atribut integrity saat mengambil skrip dari sumber eksternal seperti CDN. Ini memastikan bahwa skrip yang diunduh belum dimodifikasi.

Keamanan Tambahan: Saat Anda ingin menambah lapisan keamanan ekstra untuk memastikan bahwa hanya versi tertentu dari skrip yang digunakan pada halaman web Anda.

  • << script src="myscripts.js" referrerpolicy="origin">< /script>

Atribut referrerpolicy pada tag script> dalam HTML digunakan untuk mengontrol informasi referer yang dikirim bersama dengan permintaan skrip. Ini memungkinkan pengembang untuk menentukan seberapa banyak informasi tentang URL sumber (halaman yang memuat skrip) yang dikirimkan ke server ketika skrip eksternal diambil.

Dengan menggunakan atribut referrerpolicy, Anda dapat mengontrol lebih baik bagaimana dan kapan informasi referer dikirimkan saat mengambil skrip eksternal, meningkatkan privasi dan keamanan aplikasi web Anda.

  1. no-referrer: Tidak mengirimkan informasi referer.
  2. no-referrer-when-downgrade: Mengirimkan informasi referer kecuali jika permintaan adalah dari HTTPS ke HTTP (default)
  3. origin: Mengirimkan hanya origin dari dokumen yang memuat (misalnya, mengirim https://example.com dari https://example.com/page.html).
  4. origin-when-cross-origin: Mengirimkan URL penuh saat permintaan bersifat same-origin, dan hanya origin saat permintaan bersifat cross-origin.
  5. same-origin: Mengirimkan referer penuh untuk permintaan same-origin dan tidak mengirimkan informasi referer untuk permintaan cross-origin.
  6. strict-origin: Mengirimkan hanya origin saat protokol keamanan sama (HTTPS→HTTPS), dan tidak mengirimkan referer saat protokol berbeda (HTTPS→HTTP).
  7. strict-origin-when-cross-origin: Mengirimkan referer penuh saat same-origin dan hanya origin saat cross-origin, serta tidak mengirimkan referer saat downgrade (HTTPS→HTTP).
  8. unsafe-url: Mengirimkan referer penuh dalam semua permintaan, tanpa memperhatikan protokol.
  • << script src="myscripts.js">< /script>

Atribut src pada tag script> dalam HTML digunakan untuk merujuk ke file eksternal yang berisi kode JavaScript. Dengan menggunakan atribut ini, Anda dapat memisahkan kode JavaScript dari dokumen HTML, yang membantu dalam pengorganisasian dan pemeliharaan kode.

src: Atribut ini memerlukan URL yang menunjukkan lokasi file JavaScript eksternal yang ingin Anda masukkan ke dalam halaman HTML.

  • << script type="text/javascript"> document.getElementById("demo").innerHTML = "Hello JavaScript!"; < /script>

Atribut type pada tag script> dalam HTML digunakan untuk menentukan jenis konten skrip. Nilai atribut ini biasanya merupakan tipe MIME (Multipurpose Internet Mail Extensions) yang menunjukkan bahasa atau format dari skrip tersebut. Atribut type umumnya digunakan untuk mengindikasikan bahwa skrip tersebut adalah JavaScript, tetapi juga bisa digunakan untuk jenis skrip lain atau bahkan untuk skrip template.

  1. << script type="text/javascript" src="script.js"> < /script>

    text/javascript: Nilai default untuk skrip JavaScript. Secara historis, ini adalah cara umum untuk menentukan JavaScript, meskipun sekarang dianggap opsional karena HTML5 memperlakukan JavaScript sebagai default.

  2. << script type="module" src="module-script.js"> < /script>

    module: Digunakan untuk mengindikasikan bahwa skrip adalah modul ES6. Modul ES6 mendukung impor dan ekspor di JavaScript, memungkinkan modularisasi kode.

  3. << script type="application/ld+json">
    {
    "@context": "https://schema.org",
    "@type": "Organization",
    "name": "Example Inc.",
    "url": "https://www.example.com"
    }
    < /script>

    application/ld+json: Digunakan untuk skrip yang berisi data JSON-LD untuk pengkodean data terstruktur.

  4. << script type="text/babel"> // kode JSX di sini < /script>

    text/babel: Digunakan untuk skrip yang ditulis dalam sintaks JSX (React). Ini sering digunakan dengan Babel, sebuah alat yang mengubah kode JSX menjadi JavaScript yang dapat dijalankan oleh browser.

  5. << script type="text/template" id="template1">
    < div>
    < h1>{{title}}< /h1>
    < p>{{description}}< /p>
    < /div>
    < /script>

    text/template: Digunakan untuk skrip template yang biasanya akan diproses oleh JavaScript lain di halaman.

Menggunakan type

JavaScript Default: Untuk skrip JavaScript biasa, tidak perlu menambahkan type="text/javascript" karena ini adalah nilai default di HTML5.

Modul JavaScript: Gunakan type="module" jika Anda menggunakan fitur modul ES6.

Skrip Khusus: Gunakan tipe MIME yang sesuai jika Anda menulis skrip dalam format atau bahasa lain seperti JSON-LD atau JSX.

Template Skrip: Gunakan type="text/template" atau nilai khusus lain untuk skrip template yang akan diproses oleh JavaScript.

Element noscript

Tag noscript> dalam HTML digunakan untuk menyediakan konten alternatif yang akan ditampilkan kepada pengguna yang memiliki JavaScript dinonaktifkan di browser mereka atau jika browser mereka tidak mendukung JavaScript. Ini memungkinkan pengembang web untuk memastikan bahwa halaman mereka tetap dapat diakses dan fungsional meskipun JavaScript tidak tersedia.

Misalnya, Anda dapat menggunakan tag noscript> untuk menampilkan pesan kepada pengguna yang memberi tahu mereka bahwa JavaScript diperlukan untuk melihat konten tertentu:

<< script> document.write("Hello World!")< /script>

<< noscript>Your browser does not support JavaScript!< /noscript>

Menggunakan Tag noscript:

Pesan Pemberitahuan: Untuk memberi tahu pengguna bahwa halaman web memerlukan JavaScript untuk fungsionalitas penuh.

Konten Alternatif: Untuk menyediakan alternatif statis atau fallback ketika fitur interaktif yang memerlukan JavaScript tidak dapat digunakan.

SEO dan Aksesibilitas: Untuk memastikan bahwa konten penting tetap dapat diakses oleh pengguna dan mesin pencari meskipun JavaScript dinonaktifkan.

Element embed

Tag embed> dalam HTML digunakan untuk menyematkan konten eksternal dalam dokumen HTML, seperti video, audio, dokumen PDF, atau aplikasi interaktif. Tag ini mendukung berbagai jenis media dan format file, dan memungkinkan browser untuk menampilkan konten tersebut secara langsung dalam halaman web.

Element object

Tag object> dalam HTML digunakan untuk menyematkan objek eksternal ke dalam dokumen HTML. Ini bisa berupa berbagai jenis konten seperti gambar, audio, video, animasi Flash, atau bahkan aplikasi interaktif seperti Java applets.



Element param

Tag param> dalam HTML digunakan untuk menentukan parameter-parameter yang diperlukan oleh elemen object> atau applet>. Tag ini biasanya digunakan dalam konteks menyematkan aplikasi atau objek yang memerlukan parameter khusus untuk dikonfigurasi.

Tag param> adalah tag kosong, yang berarti tidak memiliki tag penutup, dan hanya menggunakan atribut untuk menentukan parameter-parameter.

Attribut param

  • < param name="parameter_name" value="parameter_value">

Atribut name pada tag param> dalam HTML digunakan untuk menentukan nama dari parameter yang akan dikonfigurasi untuk aplikasi atau objek yang disematkan menggunakan tag object> atau applet>.

  • < param name="parameter_name" value="parameter_value">

Atribut value pada tag param> dalam HTML digunakan untuk menentukan nilai dari parameter yang ditentukan oleh atribut name. Ini memungkinkan Anda untuk memberikan nilai yang spesifik untuk parameter yang diperlukan oleh aplikasi atau objek yang disematkan menggunakan tag object> atau applet>.
