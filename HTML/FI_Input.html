Digunakan untuk menentukan URL atau endpoint ke mana data formulir akan dikirim ketika tombol formulir diklik.

Contoh anda memiliki formulir yang menggunakan URL aksi default, tetapi ingin mengirimkan data ke URL yang berbeda ketika tombol khusus dalam formulir tersebut diklik. Anda dapat menggunakan formaction pada tombol tersebut



  • < form action="/submit" method="post" enctype="application/x-www-form-urlencoded" > < /form>
    < form action="/submit" method="post" enctype="multipart/form-data"> < /form>
    < form action="/submit" method="post" enctype="text/plain"> < /form>

Ini sudah ada di dalam attribut form sebelumnya, ini sebagai tambahan agar lebih memahami fungsi attribut ini

Digunakan untuk menentukan jenis encoding yang akan digunakan ketika data formulir dikirim ke server.

Encoding adalah proses mengonversi data menjadi format yang sesuai untuk pengiriman melalui jaringan, seperti URL encoding atau multipart/form-data untuk mengirim file.

Default (application/x-www-form-urlencoded): Ini adalah nilai default untuk formenctype. Data formulir dikodekan dalam format URL-encoded dan dikirim dalam badan permintaan

Multipart/form-data: Digunakan ketika formulir mengandung file upload. Data formulir dikirim dalam format multipart MIME.

Text/plain: Data formulir dikirimkan dalam format teks polos, tanpa pengkodean tambahan. Ini berguna untuk menguji atau debugging.



  • < form action="/submit" method="post" formmethod="post">< /form> < form action="/submit" method="get" formmethod="get" >< /form>

Lihat juga penjelasan di dalam attribut form

Digunakan untuk menentukan metode HTTP yang akan digunakan ketika data formulir dikirim ke server. Metode HTTP yang umum digunakan adalah POST dan GET.

POST:Metode ini digunakan ketika Anda ingin mengirim data formulir ke server dengan memasukkan data ke dalam badan permintaan HTTP. Ini cocok untuk pengiriman data yang sensitif atau besar.

GET:Metode ini digunakan ketika Anda ingin mengirim data formulir sebagai bagian dari URL, yang terlihat oleh pengguna. Ini cocok untuk permintaan yang dapat disematkan dan dapat dibagikan.
