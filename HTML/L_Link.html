Element link

Dalam HTML digunakan untuk mendefinisikan hubungan antara dokumen HTML dan sumber daya eksternal. Elemen ini paling sering digunakan untuk menautkan ke stylesheet, tetapi juga dapat digunakan untuk berbagai jenis hubungan lainnya, seperti ikon, prefetching, dan banyak lagi.

Elemen link> sangat berguna untuk menautkan dokumen HTML ke berbagai sumber daya eksternal, termasuk stylesheet, ikon, font, dan prefetching/prerendering. Atribut-atribut seperti rel, href, type, dan media memungkinkan Anda untuk mengontrol cara sumber daya tersebut digunakan dan ditampilkan.

< head>
< title>Document< /title>
< !-- Link to stylesheet -->
< link rel="stylesheet" href="styles.css">
< !-- Link to favicon -->
< link rel="icon" href="favicon.ico" type="image/x-icon">
< !-- Link to Google Fonts -->
< link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Open+Sans">
< !-- Prefetching a resource -->
< link rel="prefetch" href="https://example.com/resource">
< !-- Prerendering a page -->
< link rel="prerender" href="https://example.com/page">
< /head>

Attribut link

  • < link rel="stylesheet" href="styles.css">

Atribut href pada elemen link> digunakan untuk menentukan URL dari sumber daya eksternal yang ditautkan oleh dokumen HTML. Atribut ini penting karena memberitahu browser di mana menemukan sumber daya yang ingin Anda sertakan dalam halaman web Anda.

href pada elemen link> adalah kunci untuk menautkan dokumen HTML Anda ke sumber daya eksternal seperti stylesheet, ikon, font, dan sumber daya lainnya. Ini memungkinkan Anda untuk memisahkan konten dan presentasi serta meningkatkan kinerja dan fungsionalitas halaman web Anda dengan memanfaatkan sumber daya eksternal.

  • < link rel="alternate" href="https://www.example.com/en" hreflang="en">

Digunakan untuk menentukan bahasa utama dari dokumen yang ditautkan. Ini sangat berguna ketika Anda memiliki situs web multibahasa dan ingin menunjukkan kepada mesin pencari dan pengguna bahasa apa yang digunakan dalam dokumen yang ditautkan.

Sangat berguna untuk situs web multibahasa, membantu mesin pencari dan pengguna memahami bahasa utama dari dokumen yang ditautkan. Dengan menggunakan atribut ini, Anda dapat meningkatkan SEO, pengalaman pengguna, dan menghindari masalah duplikasi konten.

  • < link rel="stylesheet" href="print.css" media="print">

Digunakan untuk menentukan jenis perangkat atau kondisi media queries di mana sumber daya yang ditautkan akan diterapkan. Ini memungkinkan Anda untuk mengoptimalkan halaman web Anda dengan mengatur kapan dan bagaimana sumber daya seperti stylesheet diterapkan berdasarkan perangkat atau ukuran layar pengguna.

Sangat berguna untuk membuat halaman web yang responsif dan dioptimalkan. Dengan menggunakan atribut ini, Anda dapat mengontrol kapan dan bagaimana sumber daya seperti stylesheet diterapkan berdasarkan perangkat atau kondisi media tertentu, meningkatkan kinerja dan pengalaman pengguna.

  • < link rel="stylesheet" href="styles.css" referrerpolicy="no-referrer">

Digunakan untuk mengontrol informasi referer yang dikirim bersama permintaan yang dibuat untuk memuat sumber daya yang ditentukan dalam elemen link>. Ini sangat berguna untuk meningkatkan privasi pengguna dan mengontrol data referer yang dikirim ke server tujuan.

Atribut referrerpolicy pada elemen link> memberikan kontrol tambahan atas informasi referer yang dikirimkan bersama permintaan sumber daya. Dengan menggunakan atribut ini, Anda dapat meningkatkan privasi pengguna dan menentukan kebijakan referer yang sesuai dengan kebutuhan keamanan aplikasi Anda.

  1. no-referrer: Tidak mengirimkan informasi referer sama sekali.
  2. no-referrer-when-downgrade: Nilai default. Mengirimkan informasi referer ke target yang menggunakan HTTPS, tetapi tidak mengirimkan informasi referer ke target yang menggunakan HTTP.
  3. origin: Mengirimkan referer tanpa jalur spesifik, hanya origin (misalnya, hanya https://example.com).
  4. origin-when-cross-origin: Mengirimkan referer lengkap untuk permintaan same-origin, tetapi hanya mengirimkan origin untuk permintaan cross-origin.
  5. same-origin: Mengirimkan referer lengkap untuk permintaan same-origin, tetapi tidak mengirimkan informasi referer untuk permintaan cross-origin.
  6. strict-origin: Mengirimkan referer origin hanya untuk permintaan HTTPS, dan tidak mengirimkan referer untuk permintaan dari HTTPS ke HTTP.
  7. strict-origin-when-cross-origin: Mengirimkan referer lengkap untuk permintaan same-origin, origin untuk permintaan cross-origin HTTPS, dan tidak mengirimkan referer untuk permintaan dari HTTPS ke HTTP.
  8. unsafe-url: Mengirimkan referer lengkap untuk semua permintaan (mengabaikan protokol keamanan).
Document
  • < link rel="stylesheet" href="styles.css">

Atribut rel pada elemen link> menentukan hubungan antara dokumen saat ini dan sumber daya yang ditautkan. Ini adalah atribut penting yang digunakan untuk berbagai keperluan, seperti menautkan stylesheet, ikon, dokumen terkait, dan banyak lagi.

  1. stylesheet: Menautkan ke file stylesheet CSS yang digunakan untuk menentukan tampilan dokumen.

    2. < link rel="stylesheet" href="styles.css">

  2. icon: Menautkan ke ikon situs (favicon).

    3. < link rel="icon" href="favicon.ico" type="image/x-icon">

  3. alternate: Menyediakan alternatif untuk dokumen saat ini, seperti versi cetak atau bahasa lain.

    4. < link rel="alternate" href="print-version.html" media="print"> < link rel="alternate" href="es-version.html" hreflang="es">

  4. manifest: Menautkan ke file manifest untuk aplikasi web.

    5. < link rel="manifest" href="/manifest.json">

  5. preload: Menginstruksikan browser untuk memuat sumber daya lebih awal untuk meningkatkan kinerja.

    6. < link rel="preload" href="styles.css" as="style">

  6. prefetch: Menunjukkan kepada browser untuk mengambil sumber daya yang mungkin dibutuhkan di masa depan.

    7. < link rel="prefetch" href="next-page.html">

  7. dns-prefetch: Meminta browser untuk melakukan resolusi DNS sebelumnya.

    8. < link rel="dns-prefetch" href="//example.com">

  8. preconnect: Meminta browser untuk memulai koneksi lebih awal tanpa mengirimkan data.

    9. < link rel="preconnect" href="//example.com">

  9. canonical: Menunjukkan URL kanonis dari dokumen saat ini.

    10. < link rel="canonical" href="https://www.example.com/current-page">

  10. author: Menunjukkan bahwa URL berisi informasi tentang penulis dokumen.

    11. < link rel="author" href="humans.txt">

  11. license: Menunjukkan bahwa URL berisi informasi tentang lisensi dokumen.

    12. < link rel="license" href="license.txt">

  12. me: Menautkan ke URL yang menunjukkan identitas diri penulis.

    13. < link rel="me" href="https://www.example.com/about-me">

  13. next: Menunjukkan URL dari halaman berikutnya dalam urutan dokumen.

    14. < link rel="next" href="page2.html">

  14. prev: Menunjukkan URL dari halaman sebelumnya dalam urutan dokumen.

    15. < link rel="prev" href="page1.html">

  15. nofollow: Memberitahu mesin pencari untuk tidak mengikuti tautan ini.

    16. < link rel="nofollow" href="https://www.example.com/unimportant-page">

  • < link rel="icon" href="favicon.svg" sizes="any" type="image/svg+xml">

Digunakan terutama ketika menautkan ke ikon situs (favicon) atau ikon aplikasi untuk berbagai perangkat dan ukuran layar. Ini menentukan ukuran ikon yang ditautkan sehingga browser dapat memilih ukuran yang paling sesuai untuk digunakan.

Atribut sizes pada elemen link> memberikan informasi kepada browser tentang ukuran ikon yang ditautkan, sehingga browser dapat memilih ikon yang paling sesuai berdasarkan konteks penggunaan dan ukuran layar. Dengan menyediakan berbagai ukuran ikon, Anda dapat memastikan ikon situs Anda tampil optimal di berbagai perangkat dan resolusi layar.

  • < link rel="stylesheet" href="styles.css" type="text/css">

Digunakan untuk menentukan tipe MIME dari sumber daya yang ditautkan. Ini membantu browser memahami jenis konten yang diharapkan dari sumber daya yang ditautkan, sehingga dapat menangani sumber daya tersebut dengan cara yang benar.

Atribut type biasanya digunakan dengan sumber daya seperti stylesheet, ikon, dan lainnya untuk menentukan jenis konten yang akan diambil. Format nilai untuk atribut type adalah tipe MIME (Multipurpose Internet Mail Extensions).

Atribut type pada elemen link> memberikan informasi penting tentang tipe MIME dari sumber daya yang ditautkan. Ini memungkinkan browser untuk menangani sumber daya tersebut dengan benar, memastikan bahwa stylesheet, ikon, dan file manifest dapat dimuat dan digunakan sebagaimana mestinya. Penggunaan yang tepat dari atribut type dapat membantu mengoptimalkan kinerja dan kompatibilitas situs web Anda.
